Conceitos Glade

Glade é uma ferramenta de design de interface de usuário. It lets us use a graphical interface to design our graphical interface. Ela nos permite utilizar uma interface gráfica para projetar nossa própria interface gráfica. We could build up the window components using a bunch of calls to GTK+ functions, but it is usually easier to do this with Glade. Poderíamos construir os componentes do Windows com um monte de chamadas para funções do GTK+, mas com o Glade, é mais fazer isso.

The fundamental "thing" we work with in GTK+ is the widget . A coisa fundamental que trabalhamos com o GTK+ é o widget. A widget represents any part of the GUI, and may contain other widgets. Um widget representa qualquer parte do GUI, e pode conter outros widgets. Some examples of widgets include a window, dialog box, button, and text within the button. Alguns exemplos de widgets incluem uma janela, caixa de diálogo, botões e texto dentro do botão.

Portanto o Glade é uma ferramenta de layout widget. We set up a whole tree of widgets, with top-level windows at the top of the tree. You can think of Glade and widgets in somewhat the same terms as HTML: you can arrange widgets in a table-like layout, set up padding rules, and structure the entire description in a hierarchical way. Montamos uma árvore inteira de widgets, com as janelas de nível superior no topo da árvore. Você pode pensar no Glade e widgets em algo parecido com HTML: você pode organizar os widgets num layout de table-like, estabelecer regras de preenchimento e estruturar toda a descrição de uma maneira hierárquica.

Glade saves the widget descriptions into an XML file. O Glade salva as descrições dos widgets em um arquivo XML. Our program loads this XML file at runtime. Nosso programa carrega o arquivo XML em tempo de execução. We load the widgets by asking the Glade runtime library to load a widget with a specific name. Nós carregamos os widgets, pedindo a biblioteca runtime do Glade para carregar um widget com um nome específico.

Programação Orientada a Eventos

 GTK+, como vários pacotes de ferramentas GUI, é orientado a evento. That means that instead of, say, displaying a dialog box and waiting for the user to click on a button, we instead tell gtk2hs what function to call if a certain button is clicked, but don't sit there waiting for a click in the dialog box. Isso significa que, em vez de exibirmos uma caixa de diálogo e esperar que o usuário clique em um botão, dizemos ao gtk2hs qual função chamar, se um determinado botão é clicado, mas sem esperar por um clique na caixa de diálogo.

This is different from the model traditionally used for console programs.        Isso é diferente do modelo tradicionalmente usado para programas de console. When you think about it, though, it almost has to be. Quando você pensa nisso, porém, quase tem de ser. A GUI program could have multiple windows open, and writing code to sit there waiting for input in the particular combination of open windows could be a complicated proposition. Um programa GUI pode ter várias janelas abertas, e escrever código esperando por uma entrada em combinação particular de janelas abertas poderia ser uma proposição complicada.

Event-driven programming complements Haskell nicely.Programação orientada a eventos complementa Haskell muito bem.As we've discussed over and over in this book, functional languages thrive on passing around functions. Como já dissemos várias vezes neste livro, as linguagens funcionais crescem na passagem em torno de funções. So we'll be passing functions to gtk2hs that get called when certain events occur. These are known as callback functions .Então, passaremos as funções para gtk2hs que são chamados quando determinados eventos ocorrem. Estes são conhecidos como funções de callback.

At the core of a GTK+ program is the main loop . O núcleo de um programa GTK+ é o loop principal. This is the part of the program that waits for actions from the user or commands from the program and carries them out. Esta é a parte do programa que espera por ações do usuário ou comandos do programa e executa-os. The GTK+ main loop is handled entirely by GTK+. O loop principal do GTK+ é tratado exclusivamente pelo GTK+. To us, it looks like an I/O action that we execute, that doesn't return until the GUI has been disposed of. Para nós, parece como uma ação de E/S que executamos, na qual não retorna até que o GUI foi eliminado.

Since the main loop is responsible for doing everything from handling clicks of a mouse to redrawing a window when it has been uncovered, it must always be available.  Uma vez que o loop principal é responsável por fazer tudo dos cliques de manipulação do mouse para redesenhar uma janela quando foi descoberto, ele deve estar sempre disponível. We can't just run a long-running task -- such as downloading a podcast episode -- from within the main loop. Nós não podemos apenas executar uma tarefa de longa duração - como baixar um episódio de podcast - de dentro do loop principal. This would make the GUI unresponsive, and actions such as clicking a Cancel button wouldn't be processed in a timely manner. Isso tornaria a GUI indiferente, e ações como clicar em um botão Cancelar não seriam processados em tempo hábil.

Therefore, we will be using multithreading to handle these long-running tasks.Portanto, usaremos multithreading para lidar com essas tarefas de execução longa. More information on multithreading can be found in Chapter 24, Concurrent and multicore programming . Mais informações sobre multithreading podem ser encontradas no Capítulo 24: Programação Concorrente e multicore. For now, just know that we will use forkIO to create new threads for long-running tasks such as downloading podcast feeds and episodes. Por enquanto, usaremos o forkIO para criar novos tópicos para tarefas de execução longa, como baixar episódios podcasts e feeds. For very quick tasks, such as adding a new podcast to the database, we will not bother with a separate thread since it will be executed so fast the user will never notice. Para tarefas muito rápidas, como adicionar um novo podcast para o banco de dados, não nos preocuparemos com um segmento separado, uma vez que será executado tão rápido que o usuário nunca vai notar.

Usando o Cabal

 Nós apresentamos um arquivo Cabal para construir este projeto para uma versão de linha de comando na seção chamada "Main Program". We need to make a few tweaks for it to work with our GUI version. First, there's the obvious need to add the gtk2hs packages to the list of build dependencies. Nós precisamos fazer alguns ajustes para que funcione com a nossa versão GUI. Primeiro, há a necessidade óbvia de adicionar os pacotes gtk2hs à lista de dependências de construção. There is also the matter of the Glade XML file. Há também a questão do arquivo XML Glade.

Earlier, we wrote a PodLocalMain.hs that simply assumed this file was named podresources.glade and stored in the current working directory. Anteriormente, escrevemos um PodLocalMain.hs que simplesmente assumiu que este arquivo foi nomeado como podresources.glade e armazenados no diretório de trabalho atual. For a real, system-wide installation, we can't make that assumption. Para um sistema, no nível de instalação real, não podemos fazer essa suposição. Moreover, different systems may place the file at different locations. Além disso, diferentes sistemas podem colocar o arquivo em locais diferentes.

Cabal provides a way around this problem. Cabal fornece uma maneira de contornar este problema. It automatically generates a module that exports functions that can interrogate the environment. Ele gera automaticamente um módulo que exporta funções que podem interrogar o meio ambiente. We must add a Data-files line to our Cabal description file. Devemos acrescentar ainda uma linha do Data-files no nosso arquivo de descrição de Cabal. This file names all data files that will be part of a system-wide installation.
